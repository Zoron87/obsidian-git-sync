В **Developer Command Prompt for VS** / PowerShell:

```powershell
dotnet nuget add source "https://git.lan/repository/nuget/index.json" `
  --name "Git" `
  --username "YOUR_LOGIN" `
  --password "YOUR_TOKEN_OR_PASSWORD" `
  --valid-authentication-types basic

```

Без `--store-password-in-clear-text` пароль будет сохранён **зашифрованно (Windows)** и будет работать только для этого пользователя/машины — это нормально и безопаснее.

`--valid-authentication-types basic` полезно, если прокси/сервер “рекламирует” Negotiate/NTLM, но вам нужен именно Basic (частый случай с токенами).

### Вариант B: nuget.config + переменная окружения (удобно для CI)

1. В репозитории оставляете **только source** (без паролей).
2. Задаёте переменную окружения (на машине/в CI):
    
- Имя: `NuGetPackageSourceCredentials_Nexus`
- Значение: `Username=YOUR_LOGIN;Password=YOUR_TOKEN;ValidAuthenticationTypes=Basic`
    

Формат переменной — строго такой. https://learn.microsoft.com/ru-ru/nuget/consume-packages/consuming-packages-authenticated-feeds

### Вариант C (не рекомендую): `ClearTextPassword` в nuget.config

Возможен, но риск утечки высокий. Синтаксис `packageSourceCredentials` и предупреждения — в официальной документации.
