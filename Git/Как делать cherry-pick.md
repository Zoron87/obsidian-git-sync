**Cherry-pick** (дословно «выбирание вишенки») — это мощный инструмент Git, который позволяет взять **один конкретный коммит** из одной ветки и «переклеить» его в другую, не забирая при этом все остальные изменения.

# Инструкция:

### Шаг 1: 
Подготовка Прежде чем что-то переносить, нужно убедиться, что твоя локальная копия проекта знает обо всех последних изменениях на сервере. 1. Открой терминал в папке проекта. 2. Выполни команду: ```bash git fetch --all ```

### Шаг 2: 
Найди ID нужного коммита (Hash) У каждого коммита есть уникальный «паспорт» — длинный набор цифр и букв (например, `a1b2c3d4`). 
	Перейди в ветку, где сейчас лежит твой готовый код (например, `develop`): 
```bash 
git checkout develop 
``` 
	Посмотри историю последних изменений: 
```bash
git log --oneline -n 10 
``` 
	Найди строку со своим коммитом и **скопируй первые 7-8 символов** его хеша. 
*Пример: ты видишь `e7f8g9h Фикс скрипта экспорта`. Твой хеш — `e7f8g9h`.*

P.S. Убедись, что используется хеш именно коммита, а не мерджа.
