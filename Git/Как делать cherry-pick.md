**Cherry-pick** (дословно «выбирание вишенки») — это мощный инструмент Git, который позволяет взять **один конкретный коммит** из одной ветки и «переклеить» его в другую, не забирая при этом все остальные изменения.

# Инструкция:

### Шаг 1: Подготовка 
Прежде чем что-то переносить, нужно убедиться, что твоя локальная копия проекта знает обо всех последних изменениях на сервере. 
1. Открой терминал в папке проекта. 
2. Выполни команду: 
```bash
git fetch --all 
```


## Часть 2: Создание ветки для переноса 
2.1. Создай новую ветку от "Цели" 
Нам нужно создать временную ветку, основанную на той ветке, **куда** мы хотим в итоге положить код (в `release_v2.0.1`).
```bash 
# Переходим на целевую релизную ветку 
git checkout release/release_v2.0.1 

# Обновляем её до состояния сервера 
git pull origin release/release_v2.0.1 

# Создаем от неё новую ветку для PR (назови её по номеру задачи) 
git checkout -b fix/895640-compliance-fix 
``` 

*Теперь ты стоишь в новой ветке, которая является точной копией релиза.*

### Шаг 2: Найди ID нужного коммита (Hash) 
У каждого коммита есть уникальный идентификатор — длинный набор цифр и букв (например, `a1b2c3d4`). 
	Перейди в ветку, где сейчас лежит твой готовый код (например, `release_v2.0.1`) или release/release_v2.0.1: 
```bash 
git checkout release_v2.0.1 
``` 
	Посмотри историю последних изменений: 
```bash
git log --oneline -n 10 
``` 
	Найди строку со своим коммитом и **скопируй первые 7-8 символов** его хеша. 
*Пример: ты видишь `e7f8g9h Фикс скрипта экспорта`. Твой хеш — `e7f8g9h`.*

P.S. Убедись, что используется хеш именно коммита, а не Merge-Commit (иначе git не поймет из какой ветки брать изменения).

### Шаг 3: Перейди в целевую ветку. 
Теперь тебе нужно встать на ту ветку, **куда** ты хочешь добавить изменения. 
В нашем случае это релизная ветка. 
1. Переключаемся: 
```bash
git checkout release_v2.0.1 
``` 
2. Обнови её до последней версии с сервера: 
```bash 
git pull origin release_v2.0.1 
```

### Шаг 4: Делаем Cherry-pick 
 1. Выполни команду: 
 ```bash   
 git cherry-pick e7f8g9h 
 ```

 (Вместо `e7f8g9h` вставь свой хеш, который скопировал на Шаге 2)

### Шаг 5: Отправка на сервер
Коммит теперь приклеен локально к ветке `release_v2.0.1`. 
Осталось отправить его в репо. 
1. Выполни: 
```bash 
git push origin release_v2.0.1
```

# FAQ
1. Отмена cherry-pick
```bash
git cherry-pick --abort
```
