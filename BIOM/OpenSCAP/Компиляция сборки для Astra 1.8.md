Для того, чтобы скомпилировать доработанную сборку OpenSCAP 1.3.7 под Astra 1.8 необходимо:

1. В файле CMakeLists на строке 53 изменить набор требуемых пакетов (т.к. Conan 1 больше не поддерживается и часть пакетов не скачивается при компиляции)
```c++
set(CONAN_PACKAGES_LIST boost/1.86.0
                        libxml2/2.13.4
                        openssl/3.3.2
                        libiconv/1.17
                        libxslt/1.1.42
                        zlib/1.3.1
                        pcre/8.45
                        libssh/0.10.6
                        libcurl/8.10.1
)
```

2. На строке 66 удалить строку:
```c++
libssh:disable_export_symbols=True
```

т.к. в используемой версии пакета данного параметра нет

3. Также в одном из файлов забыли подключить стандартную библиотеку `limits.h`, где определены константы `INT_MIN` и `INT_MAX`. Раньше это работало потому что этот файл подключался неявно через другие файлы, но теперь это нужно делать явно.

Нужно вручную исправить файл исходного кода.

### Инструкция по исправлению:

1.  **Открой файл с ошибкой в редакторе:**
    (Ты сейчас находишься в папке `build`, поэтому путь указываем через `../`)

    ```bash
    nano ../src/OVAL/results/oval_cmp_evr_string.c
    ```

2.  **Добавь недостающую строку:**
    Найди в самом начале файла блок, где идут строки `#include`.
    Добавь туда строку:
    `#include <limits.h>`
